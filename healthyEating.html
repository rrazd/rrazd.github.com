Healthy Eating Application UX and Front End Development

Background

The Healthy Eating application is a dynamic web application that users can utilize to keep track of the amount of calories that they are consuming. This task is made more interesting by the fact that this application interacts with Facebook to link the user with all of their friends who also use the application for keeping track of calories. Users can compete with their friends in order to determine who is superior in terms of eating healthy.

Development

To create the application Google Web Toolkit was used (GWT). Using this framework made it easier to implement design patterns and use Java and Eclipse to create the final product. Furthermore, the framework ensured the presence of cross browser compatibility and other non-functional ‘nice to have’ features.
The application had some notable technical complexity. It utilized:

  • Remote Procedure calls to communicate with MySQL database
  • BCrypt framework for login password hashing
  • Facebook API
  • Google Web Toolkit Visualizations API
  • Google Web Tookit Framework

Testing

In order to see if there was interest in the application's services a prototype that provided the very basic simplified features was created. Java Servlets, Java Server Pages and MySQL were used. The pertinent screenshots are shown below: A sample user group was asked to play around with the application and report back on if they found its features useful. The feedback suggested that certain features such as the visualization of a user's calorie intake during the day was heavily used. In contrast, other features such as the social media component where users could compare their progress with their friend's were infrequently used and needed improvement.

Conclusion

In conclusion, the final product was a fully functioning web application that integrated social media to increase user engagement.
